關鍵決策與發現

導覽架構：採設計稿的「5-mode ＋ Preview dock ＋ Settings modal」

Spec 期望七個入口，但設計稿實際是 sidebar 5-mode switcher（Files/Git/Database/SSH/Agent）＋ Preview 為 editor 工具列 dock 切換 ＋ Settings 為 rail 頭像 modal。裁決採設計稿架構——與 spec「docked browser panel」的 Live Preview 定義一致，七個入口全數保留、只是層次不同。其餘 8 條架構裁決（lime accent、dark 用 .yzdark 值、266px sidebar、IDE 字級 scale、@fontsource、empty-state 基準、SSH/SFTP 同入口、primary＝深色實心軌）詳見 .superpowers/sdd/progress.md。Final reviewer 確認 9 條決策在 6 個獨立 subagent 任務間零漂移。

與既有 plan（foundation-logs-shell）的偏離

依你本次指示：Task 6 的 plain CSS 改為 shadcn＋design tokens；logs 相關（plan Tasks 2–5、7）不在本階段，useUserActionLog 以 no-op stub 佔位（介面已定型為 async，之後可直接接 Tauri command）；Rust log 依賴延後；所有 commit 步驟因 Git 安全規則略過。

與 M1 Agent Loop 並行工作流的協調

同 repo 有另一個 session 跑「M1 editor core」agent loop。雙方透過共用 ledger 協調出檔案所有權：UI Phase 持有 src/app/** 與 useUserActionLog；M1 持有 package.json／Cargo.toml／.gitignore 等。M1 的 editor core 將掛進 <EditorPanel />。過程中一次事故：E1 implementer 用 pkill -f vite 誤殺 M1 spike 的 dev server（約 1 分鐘後恢復、檔案未受影響），已通報並改為 PID 精準管理。

設計 handoff 的品質問題（回饋給設計流程）

設計稿 tokens 與實際原型有系統性出入：原型大量覆寫 tokens.css（accent 用 lime 而非 Arc 藍、dark 覆寫 --paper-* 而非用 --night-*、sidebar 266px 而非 248px、IDE 字級完全自訂）。設計分析任務已全部以原型實際值為準並記錄 12 條歧義；多處 empty state 原型未設計，以 SQL console 樣式為基準延伸。
