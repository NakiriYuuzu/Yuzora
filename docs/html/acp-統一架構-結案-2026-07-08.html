驗證方式（範圍化 gate）

為何範圍化：執行期間你並行的 db/context-menu WIP（DatabaseNavContent.tsx、dbStore.ts、contextMenuDefs.ts 等）進入工作區且不完整，使全量 bun run build／bun run test 出現 db 相關 tsc 錯與測試失敗。這與 ACP 無關，我全程未碰你的檔。故 gate 收緊為「ACP 範圍」。

權威 gate（每個 task 都跑）

  • tsc：bunx tsc --noEmit | grep 'error TS' | grep -v <db 檔> → 須為空（抓 interface 改動波及 consumer 檔的回歸）。
  • 測試：bun run test -- <ACP 檔切片> → 全綠。最終彙總 99 個 ACP 測試通過。
結案時：ACP 範圍 tsc 零錯、99 個 ACP 測試全綠。每個 task 皆 fresh implementer → fresh reviewer → fix loop；控制端對每個 gate 親自 CLI 覆核（IDE 診斷常 stale，以 CLI 為權威）。
