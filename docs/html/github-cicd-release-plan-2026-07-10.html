整體流水線：三條 workflow 的觸發與產出

三條 workflow 各自獨立：CI 顧品質、Release 顧產物、Pages 顧產品頁。Release 產出為 draft，最後一步始終由人工 Publish。

設計原則

  • tag 驅動發布：v* tag 是唯一的發布觸發點，main 上的日常 commit 永不觸發建置發布。
  • draft-first：tauri-action 建立 draft release，三平台 artifacts 齊了、release notes 寫了，才由人工 Publish——機器不直接對外發布。
  • 單一版本真相：以 tauri.conf.json 的 version 為準；guard job 驗 tag 一致性，擋掉「tag 打了但忘記 bump」的事故。
  • 產品頁零維護（方案 A）：release workflow 會把主要安裝檔以固定檔名（如 Yuzora-macos-universal.dmg）額外上傳一份，產品頁連 releases/latest/download/<固定檔名> 即可「點了直接下載」，發新版不需要動頁面。注意：首次 Publish 之前這些直接下載連結會 404（預期行為）。
  • Pages 用 Actions source：不用 gh-pages branch，避免產物 commit 汙染歷史；只上傳 site/，Remotion 原始碼（site-remotion/）不進部署產物。

release.yml 新增

tag v* 觸發。guard 先驗版本一致，再以 matrix 三平台跑 tauri-apps/tauri-action，自動建 draft release 並上傳安裝檔。macOS 出 universal binary（Apple Silicon ＋ Intel 單一 .dmg）。

.github/workflows/release.yml
name: Release

on:
  push:
    tags: ['v*']

permissions:
  contents: write

concurrency:
  group: release-${{ github.ref }}
  cancel-in-progress: false

jobs:
  guard:
    name: Tag / version consistency
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Verify tag matches tauri.conf.json version
        run: |
          TAG="${GITHUB_REF_NAME#v}"
          VERSION=$(jq -r .version src-tauri/tauri.conf.json)
          if [ "$TAG" != "$VERSION" ]; then
            echo "::error::tag v$TAG != tauri.conf.json version $VERSION" >&2
            exit 1
          fi

  build:
    needs: guard
    strategy:
      fail-fast: false
      matrix:
        include:
          - platform: macos-latest
            args: '--target universal-apple-darwin'
            rust-targets: 'aarch64-apple-darwin,x86_64-apple-darwin'
          - platform: ubuntu-22.04
            args: ''
            rust-targets: ''
          - platform: windows-latest
            args: ''
            rust-targets: ''
    runs-on: ${{ matrix.platform }}
    steps:
      - uses: actions/checkout@v4

      - uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - uses: dtolnay/rust-toolchain@stable
        with:
          targets: ${{ matrix.rust-targets }}

      - name: Install Linux system deps
        if: matrix.platform == 'ubuntu-22.04'
        run: |
          sudo apt-get update
          sudo apt-get install -y libwebkit2gtk-4.1-dev libappindicator3-dev \
            librsvg2-dev patchelf libgtk-3-dev

      - uses: Swatinem/rust-cache@v2
        with:
          workspaces: src-tauri

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - uses: tauri-apps/tauri-action@v0
        id: tauri
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          # ---- macOS 簽章／notarization（Phase 2：補上 secrets 即自動生效）----
          # APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }}
          # APPLE_CERTIFICATE_PASSWORD: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
          # APPLE_SIGNING_IDENTITY: ${{ secrets.APPLE_SIGNING_IDENTITY }}
          # APPLE_ID: ${{ secrets.APPLE_ID }}
          # APPLE_PASSWORD: ${{ secrets.APPLE_PASSWORD }}
          # APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
        with:
          tagName: ${{ github.ref_name }}
          releaseName: 'Yuzora ${{ github.ref_name }}'
          releaseDraft: true
          prerelease: false
          args: ${{ matrix.args }}

      # 產品頁「直接下載」用的固定檔名別名（方案 A）：
      # tauri-action 的產物檔名含版本號，這裡把主要安裝檔以不變的檔名
      # 再上傳一份到同一個 draft release，讓
      # releases/latest/download/<固定檔名> 永久有效。
      - name: Upload fixed-name alias assets
        shell: bash
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          RELEASE_ID: ${{ steps.tauri.outputs.releaseId }}
          ARTIFACTS: ${{ steps.tauri.outputs.artifactPaths }}
        run: |
          echo "$ARTIFACTS" | jq -r '.[]' | while read -r f; do
            case "$f" in
              *.dmg)       alias="Yuzora-macos-universal.dmg" ;;
              *setup.exe)  alias="Yuzora-windows-x64-setup.exe" ;;
              *.msi)       alias="Yuzora-windows-x64.msi" ;;
              *.AppImage)  alias="Yuzora-linux-x86_64.AppImage" ;;
              *.deb)       alias="Yuzora-linux-amd64.deb" ;;
              *.rpm)       alias="Yuzora-linux-x86_64.rpm" ;;
              *)           continue ;;
            esac
            echo "upload $f -> $alias"
            curl -sf -X POST \
              -H "Authorization: Bearer $GITHUB_TOKEN" \
              -H "Content-Type: application/octet-stream" \
              --data-binary @"$f" \
              "https://uploads.github.com/repos/${GITHUB_REPOSITORY}/releases/${RELEASE_ID}/assets?name=${alias}" > /dev/null
          done

deploy-pages.yml 新增

main 上 site/** 有變更時部署；也可用 workflow_dispatch 手動觸發。前置：repo Settings → Pages → Source 選「GitHub Actions」。

.github/workflows/deploy-pages.yml
name: Deploy Pages

on:
  push:
    branches: [main]
    paths: ['site/**']
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/configure-pages@v5
      - uses: actions/upload-pages-artifact@v3
        with:
          path: site
      - id: deployment
        uses: actions/deploy-pages@v4

每次發版的標準流程

Step 1 · 品質門檻
確認 main 綠燈

最後一個 commit 的 CI（frontend ＋ rust）全數通過；本機 bun run build、bun run test、cargo check 亦可先行驗證。

Step 2 · 版本同步
三處版本號一起 bump

package.json → src-tauri/tauri.conf.json → src-tauri/Cargo.toml，改完跑 bun install 與 cargo check 讓 lockfile 跟上。三處必須一致，guard job 會以 tauri.conf.json 為準驗 tag。

Step 3 · commit ＋ tag
版本 commit 與 tag（需你明令）

git commit -m "chore(release): v0.2.0" → git tag v0.2.0 → git push origin main v0.2.0。tag push 即觸發 release workflow。

Step 4 · 自動建置
三平台並行建置（約 20–40 分鐘）

guard 通過後，macOS universal / Windows / Linux 各自建置並把安裝檔上傳到同一個 draft release。任一平台失敗不影響其他平台（fail-fast: false），可單獨 re-run。

Step 5 · 人工把關
檢查 draft → Publish

確認三平台 artifacts 齊全、抽驗下載安裝、撰寫 release notes（可用 Releases 頁的 generate notes），按下 Publish 才正式對外。

Step 6 · 產品頁
不需動作

site/ 的下載連結固定指向 releases/latest，Publish 後自動生效。產品頁本身只在 site/** 變更時才重新部署。

版本號管理建議

三處版本並存是 Tauri 專案的常態；規則定為「tauri.conf.json 是真相，其他兩處跟隨」。若之後嫌手動 bump 麻煩，可加一個 scripts/bump-version.ts（bun 腳本，一次改三處＋更新 lockfile），屬 nice-to-have，Phase 1 手動即可。

決策點選項與建議
Repo 可見性 gh-pages 在免費方案僅 public repo 可用（private 需 Pro）。產品頁 ＋ open source 定位 → 建議 public（頁面文案已寫「原始碼公開可查」，若要 private 需回頭調整文案與 Pages 方案）。
macOS 簽章 Phase 1 不簽：使用者首次開啟需右鍵 → 開啟（或 xattr -cr /Applications/Yuzora.app），建議在 Releases notes 與產品頁 FAQ 註明。Phase 2 申請 Apple Developer（USD 99/年）後把六個 secrets 填入，release.yml 的註解解開即自動簽章＋notarize。

自動更新（updater） 需引入 tauri-plugin-updater、簽章金鑰（TAURI_SIGNING_PRIVATE_KEY）與 latest.json endpoint，牽動 app 本體。明確列為 Phase 2，本次不納入。

風險與緩解

風險等級緩解（已納入設計）
tag 與版本號不一致，發出錯誤版本高guard job 比對 tag 與 tauri.conf.json，不一致直接 fail，不進建置
未簽章的 macOS 版被 Gatekeeper 擋，使用者以為壞檔中Release notes ＋ 產品頁註明開啟方式；Phase 2 補簽章後消失
universal build 拉長 CI 時間、吃 Actions 額度中rust-cache＋只在 tag 時建置；public repo 的 Actions 免費無上限
Pages 誤把 Remotion 原始碼、node_modules 部署上線低artifact 只上傳 site/；site-remotion/ 不在部署路徑內
機器直接對外發布出事故低releaseDraft: true，Publish 永遠人工
Linux 建置缺系統套件紅燈低workflow 內建 webkit2gtk 4.1 等 apt 相依（Tauri v2 × ubuntu-22.04 對應版本）

驗收 checklist（實作後逐項打勾）

  • gh repo create ＋ push 後，main 的 CI 綠燈
  • Pages source 設為 GitHub Actions，deploy-pages.yml 跑過一輪，產品頁可公開瀏覽（含影片、中英切換）
  • 打 v0.1.0 tag → guard 通過 → 三平台 artifacts 出現在 draft release
