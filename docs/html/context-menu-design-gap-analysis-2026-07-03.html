Context Menu：設計 vs 實作交叉驗證

設計來源：docs/design/project/Yuuzu Workbench.dc.html · 實作範圍：src/ + src-tauri/ · 2026-07-03

設計文件的 context menu 系統目前完全沒有落地。整個 src/ 對「contextmenu」的大小寫不敏感搜尋為 0 命中：沒有任何 onContextMenu handler、沒有 shadcn/Radix 的 context-menu 元件（ui 目錄只有未被引用的 dropdown-menu）、Tauri 端也沒有原生選單處理。實際效果是：在 app 任何位置按右鍵，會彈出 WKWebView 的系統預設英文選單（Reload、Copy…），連設計中「preview 面板要吃掉右鍵」的最低要求也不成立。

需要公平標註的是：M1 / M2 兩份 spec 都從未把 context menu 排入 scope，所以這是「設計已完整定義、但尚未排入任何里程碑」的規劃缺口，而不是既有里程碑的實作疏漏。

設計文件對 context menu 的完整要求

  • 統一的選單系統：單一 popup（寬 212px、項目高 31px、圓角 12px、frost 玻璃 + blur、yzpop 120ms 彈性動畫、z-index 80），由 openCtx(e, kind) 分派，ctxRun(kind, key, host) 執行動作（設計文件 L1467–1479、L3527–3562、L2054–2105）。
  • 12 種區域選單（ctxDefs）：general / rail / explorer / file / tab / editor / terminal / agent / git / status / sshhost，未知 kind 時 fallback 到 explorer。
  • 邊界處理：選單位置 clamp 在視窗內（距右、下邊界 8px）。
  • 危險動作紅字：刪除、終止程序、還原、中斷連線等以 #c2293f 呈現。
  • 動態項目：terminal 選單的停靠項會依目前 dock 狀態切換文案（「與程式碼分割」↔「停靠底部」）。
  • 兩個附加需求：project header 右鍵開啟「專案編輯 popover」（改 icon / 名稱 / 顏色，L118、L1544）；preview 面板右鍵 noCtx（preventDefault，不彈任何選單，L538）。

補齊建議（分三階段，跟著里程碑走）

階段 0 — 共用基礎（一次性）

建立單一 ContextMenu 元件 + useContextMenu store：資料驅動（沿用設計的 ctxDefs 結構）、fixed 定位 + 視窗邊界 clamp、danger 樣式、Esc / 點外關閉。可用 Radix ContextMenu（shadcn context-menu）或自製 portal — 設計的 frost 玻璃與 yzpop 動畫用自製較容易貼齊。同時在 app 根節點統一 preventDefault，先杜絕系統選單露出（含 preview 面板的 noCtx 需求）。

階段 A — M1 範圍即可接上的區域

  • general / rail：palette、settings、sidebar toggle 均已存在。
  • tab：補 store 的 closeOthers / closeAll，加右鍵（已在 7/02 gap 分析追蹤）。
  • file / explorer：先上「開啟、複製路徑／相對路徑」等零依賴項；rename / delete / 新建 / reveal 需要新增 Rust fs 指令，可另立小任務。
  • editor：剪下／複製／貼上 + 指令中心（格式化留待有 formatter 時）。

階段 B / C — 隨功能里程碑落地

  • M2（Git）：git 面板與 status 狀態列選單，建議直接寫進 M2 spec 的 UI 章節。
  • Terminal / Agent / SSH 里程碑：terminal（含動態停靠項）、agent、sshhost、project header popover 隨各自功能一起實作，避免做出一堆空殼選單項。

All Findings

Critical
全域 — 設計定義的 context menu 系統 0/12 落地；右鍵目前露出 WKWebView 系統預設英文選單，直接破壞產品一致性。建議至少先做「階段 0」的全域攔截。
Warning
規劃缺口 — M1、M2 spec 均未把 context menu 排入 scope；若不主動排程，這 13 項會持續無人認領。建議：階段 A 併入 M1 收尾或 M2、git/status 選單寫入 M2 spec。
Warning
src/workbench/TabBar.tsx — 唯一曾被追蹤的缺口（7/02 gap 分析列為「潤飾」級），但其依賴的 closeOthers / closeAll store 操作也尚未存在。
Suggestion
src/app/panels/PreviewPanel.tsx — 設計要求 preview 面板 noCtx（吃掉右鍵）。零依賴、一行 handler 可完成，適合隨手補上。
Suggestion
src/components/ui/dropdown-menu.tsx — 已引入但全專案未使用；實作 context menu 時應決定沿用 Radix 系（加裝 context-menu）或自製，避免兩套選單系統並存。
Info
設計原型限制 — ctxRun 中 rename / 新建資料夾 / reveal / duplicate 等在原型內僅為 toast stub；實作前需在 spec 補上真實行為定義（特別是檔案系統操作的錯誤處理）。
