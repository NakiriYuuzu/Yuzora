A · Ledger 帳冊閱讀流

Signature：左側 margin rail（serif 眉批）

去泡泡、全寬編輯部式閱讀流。說話者與時間退到左欄的 serif 眉批（用的是 app 內建的 Newsreader，零新字體），內容欄拿到完整寬度。tool／plan 從「卡片」降為「縮排條目」（2px 墨線），permission 保留唯一的色塊強調。串流不用方塊游標，改為最後一行的 lime 呼吸底線。

解決「醜」
真 markdown 層級（標題／清單／程式碼）＋serif 對比＝閱讀質感；視覺噪音（各種框）大幅減少。
解決「佔位」
取消氣泡 88/94% 寬度浪費；thought 收成一行；tool 一行一條目。
實作對應
純 CSS＋native blocks（無新相依）；需 markdown 升級（Phase 4 的 Response 或擴充 markdown-it）。
風險
最不像「聊天」；user 訊息辨識度靠左欄眉批，掃讀時對話感較弱。

B · Instrument 儀表事件流

Signature：thread spine（串流時 lime 脈動節點）＋ prompt echo

把 transcript 當成機器的事件流：左側一條 spine，每個事件掛一個狀態節點（像 git graph／CI log），user 輸入渲染成 ❯ prompt echo——誠實對應「這是開發工作台、旁邊就是真的 terminal」。tool 一行一列（kind＋名稱＋摘要＋耗時＋狀態），permission 是 spine 上的「中斷」條，帶鍵盤提示。

解決「醜」
mono 資訊層級＋語意色（lime=live、mint=done、amber=wait）＝儀表的秩序感。
解決「佔位」
四個方向中密度最高：tool／plan／thought 全部單行起跳、可展開。
實作對應
純 CSS＋native blocks；spine 節點即 stable-id 的視覺化，跟 Phase 0 相性最好。
風險
最冷；非技術訊息（長文回覆）在事件流裡稍顯擁擠。

C · Atelier 玻璃工作室

Signature：ambient 漸層畫布＋frost glass 分層

把 app 自己的 splash／glass 視覺語言帶進對話面：transcript 畫布用 Yuzora 的 ambient 漸層（現況是死平的 sunk 灰），user 保留玻璃氣泡（右側、backdrop-blur），agent 改為「頭像＋全寬內容」的混合式。tool／plan／permission 是懸浮的 frost 卡片，composer 是浮在漸層上的玻璃艙。

解決「醜」
材質層次（漸層×玻璃×陰影）直接繼承 app 最好看的部分（splash／rail）；品牌一致性最強。
解決「佔位」
agent 訊息全寬（只有 user 保留氣泡）；thought 一行；tool 可合併成群組卡。
實作對應
純 CSS（backdrop-filter）＋native blocks；心智模型與現況最接近、遷移最平滑。
風險
blur 疊層在長 transcript 的滾動效能需實測（WebView backdrop-filter 成本）；「玻璃」用過量會膩。

回饋

也可以混搭（例如「C 的畫布＋B 的 tool 列」）——寫在備註即可。

字體註：mockup 以系統字體近似（Hanken Grotesk／JetBrains Mono／Newsreader 未安裝時分別退回系統 sans／SF Mono／Georgia）；實作時 app 內三套字體皆已內建（@fontsource-variable）。色票全部取自 src/styles.css 的實際 token（light＋dark 兩套）。四欄內容語意完全相同，高度數字為 transcript 區 scrollHeight 實測。互動（展開／折疊／hover）為靜態示意，選定方向後於 spec 定義。
