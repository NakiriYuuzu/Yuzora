SSH 真 TOFU ssh_service.rs:149-175

check_server_key 三分支：Changed → Ok(false)（russh 收到後在 initial kex 期間回 UnknownKey 中止握手，認證封包在此之後才送 → 憑證絕不外洩給變更的 host）；New → 持久化到 ~/.yuzora/known_hosts.json 後 Ok(true)（首見信任）；Match → Ok(true)。殘留缺口（首見無 pre-auth 人工確認）誠實記在 doc comment。覆核結論：紮實的真 TOFU，非 accept-all。

fs symlink 防護 fs_service.rs:208+

resolve_in_workspace：canonicalize root → lexical 正規化目標擋 .. → canonicalize「已存在最深祖先」確認真實位置仍在 root 底下（尾段尚不存在的用 lexical 保證無 ..）。四個 write command（fs_rename 的 from/to 皆驗）全數經過。測試建真實外部 symlink 斷言 create/rename/delete 全 fail-closed。

rename/delete stale tab contextMenuStore.ts:303-332

deleteEntry：confirm dialog（資料夾特別警告）→ fsDelete → affectedTabPaths 找出所有受影響 tab（含資料夾底下）→ closeTabsByPath 關閉 → dropDocument 清 documentRegistry（防存檔重建舊路徑）→ 清 markdown preview。renameEntry 用 updateTabPath＋renameDocument 更新而非關閉。
