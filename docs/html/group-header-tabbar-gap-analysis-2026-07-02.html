總覽

你貼的 group-header 片段，是 group.tabs.length === 0（沒有任何開啟中的分頁）時 EditorArea.tsx 的實際 render 結果，內部巢狀著 TabBar.tsx。跟設計文件第 357–391 行的 tab bar 規格對照後，最直接能在畫面上看到的落差有兩處：空狀態文字、以及右側工具列少了一顆按鈕。除此之外，TabBar 元件本身在「有分頁時」該具備的能力（preview tab、terminal tab、git modified 標記、右鍵選單、拖曳排序）目前完全沒有實作，資料模型（TabInfo）也還沒預留這些欄位。

這次畫面片段直接暴露的落差

  • 空狀態文案：目前寫死英文 "No open tabs"，設計文件是雙語 noOpenTabs（英文含提示句、中文另有版本）。
  • group-actions 只有 2 顆按鈕：設計文件在「Toggle preview」跟「Split editor」中間，還有一顆「終端機位置」按鈕（含下拉選單），目前完全沒做。

建議優先處理

  • 先補空狀態文案雙語化（影響範圍小、使用者立即可見）。
  • 終端機停靠按鈕是否要做，取決於「終端機能否停進編輯器分頁列」這個產品決策是否還要沿用設計文件的方向 — 這點建議跟你確認，而不是直接視為缺陷。
